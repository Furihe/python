# 8장 객체와 클래스
# 1. 객체지향 프로그래밍

객체지향 프로그래밍(Object-Oriented Programming, OOP)은 데이터와 기능을 하나로 묶어 프로그램을 구성하는 방법이다.

프로그램을 객체 단위로 설계한다.
코드의 재사용성과 유지보수성을 높일 수 있다.
# 2. 객체(Object)

객체는 데이터와 기능을 함께 가지고 있는 실체이다.

* 속성(Attribute) : 객체가 가지고 있는 데이터
* 메서드(Method) : 객체가 수행하는 기능

예) 자동차 객체

속성 : 색상, 속도
메서드 : 출발하기, 정지하기
# 3. 클래스(Class)

클래스는 객체를 만들기 위한 설계도이다.

객체의 구조와 기능을 정의한다.
하나의 클래스로 여러 객체를 생성할 수 있다.
# 4. 객체 생성

클래스를 이용해 객체를 생성하는 과정을 인스턴스화(Instantiation)라고 한다.

생성된 객체를 인스턴스(Instance)라고 한다.
하나의 클래스로 여러 개의 객체를 만들 수 있다.
# 5. 생성자

생성자는 객체가 생성될 때 자동으로 호출되는 특별한 메서드이다.

__init__() 메서드를 사용한다.
객체의 초기값을 설정한다.
# 6. 멤버 변수와 메서드

클래스 내부에는 데이터와 기능을 정의할 수 있다.

* 멤버 변수 : 객체의 데이터를 저장
* 메서드 : 객체의 기능을 정의
# 7. self

self는 현재 객체 자신을 가리키는 참조 변수이다.

클래스 내부에서 객체의 속성에 접근할 때 사용한다.
메서드의 첫 번째 매개변수로 작성한다.
8. 주의해야 할 점
* 클래스 이름은 일반적으로 첫 글자를 대문자로 작성한다.
* 객체를 생성해야 클래스의 기능을 사용할 수 있다.
* 메서드의 첫 번째 매개변수는 보통 self를 사용한다.
* 생성자는 __init__() 형태로 작성한다.
# 📌 핵심 정리
* 객체는 데이터와 기능을 함께 가지는 실체이다.
* 클래스는 객체를 만들기 위한 설계도이다.
* 객체를 생성하는 과정을 인스턴스화라고 한다.
* 생성자 __init__()는 객체 생성 시 자동으로 호출된다.
* self는 현재 객체 자신을 의미한다.
* 객체지향 프로그래밍은 코드의 재사용성과 유지보수성을 높여준다.
